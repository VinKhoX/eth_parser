Implement a streaming Ethernet parser with 802.1Q VLAN handling in two source files: sources/eth_parser.sv (top module) and sources/payload_extract.sv (submodule). eth_parser must instantiate payload_extract and expose all ports below. Split logic between files as you wish; the hierarchy and internal interfaces must be consistent and synthesizable.

Ports (top-level eth_parser)
clk, rst, rx_valid, rx_data[7:0], rx_err, fifo_wen[1:0], raw_payload[7:0], valid_pkt_cnt[15:0], invalid_pkt_cnt[15:0].

A) Stream semantics

While rx_valid is high, rx_data delivers one frame byte per rising clk edge, in order. rx_data is not required to hold any particular value when rx_valid is low.

End of packet (EOP) is the falling edge of rx_valid after the last byte of the current frame. The DUT must buffer the L2 payload during reception and must only begin replaying that payload after EOP (replay is not byte-for-byte in lockstep with the incoming payload stream).

B) On-the-wire frame layout (byte order)

VLAN frame: DA[6], SA[6], TPID[2] = 0x8100, TCI[2] (VID = lower 12 bits of TCI), EtherType[2], payload of N bytes, FCS[4].

Non-VLAN frame: DA[6], SA[6], EtherType[2], payload of N bytes, FCS[4].

Minimum length: a complete VLAN-tagged frame on the wire is at least 22 bytes (18-byte tag header including EtherType, plus 4-byte FCS; N may be zero). Fewer bytes received before EOP for a frame that is being interpreted as VLAN-tagged counts as truncated (invalid). Similarly, a non-VLAN frame must include at least DA, SA, EtherType, and FCS before EOP or it is truncated (invalid).

FCS: Every frame ends with 4 bytes of FCS (CRC-32, big-endian on the wire, computed over all octets before the FCS). The testbench appends this FCS. The application payload for parsing, FIFO routing, and replay is only the bytes strictly between EtherType and FCS. For VLAN frames, that is the N bytes after the 2-byte EtherType that follows TCI. The FCS octets must not be included in replay or treated as payload. Verifying the CRC is optional; excluding FCS from payload and replay is required.

C) Behavior

Valid packet (increment valid_pkt_cnt once per such frame): TPID == 0x8100, VID is 0 or 1, the frame is not truncated, and rx_err is never high while rx_valid is high during that frame. On valid frames, replay each payload byte on raw_payload. Assert fifo_wen[0] on each replay beat for VID 0 and fifo_wen[1] on each replay beat for VID 1 (one-hot per beat; bit 0 corresponds to VID 0, bit 1 to VID 1).

Invalid packet (increment invalid_pkt_cnt once per such frame): the layout is non-VLAN as in B), or rx_err is high on any cycle where rx_valid is high during the frame, or the frame is truncated as in B).

Silent drop (do not change either counter): TPID == 0x8100 and VID is greater than 1 (i.e. not 0 or 1).

D) Deliverable

The implementation must pass all tests in tests/test_eth_parser.py. Do not modify tests/test_eth_parser.py or any other file under tests/; change only the RTL under sources/.
